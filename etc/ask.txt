どんな定理かというと例を挙げると、リストの要素の平均値を求める関数は
average(lst) = sum(lst) / length(lst)と定義できますが、
sum(lst)もlength(lst)も畳み込みで定義されているので
sum(lst) / length(lst)の実行時間はリストの長さに対して線形ではありません。
しかし、Fokkingaの相互再帰定理を使って式を導出すると、
新しく作ったaverage関数はリストの長さに対して線形となるようにできる。そんな定理です。

じゃぁVimとは何か？そこから自分がわかる限り説明していこうと思います。
Vimとはテキストエディタです。テキストを編集するのに優れたソフトです。
じゃぁメモ帳とかVisualStudioとかと何が違うのかというと、
テキストを入力するとき、何を使いますか？
キーボードですよね、それ以外は殆ど無いと思います。
じゃぁテキストをコピーしたり、貼り付けたり、削除したりする時は？
マウスを使ってるって思った人いますよね？マウスでドラッグして、
右クリックして...とか、上のメニューから選んで...とか手順を踏むんじゃないかと思うんです。
でも、それって時間掛かりませんか？例えば今の行をコピーして次の行に貼り付けたいって時ありますよね。
似てるけどちょっとだけ変える時がある時にやると思います。
それをマウスでドラッグして、左とか右まで持ってって、右クリック押して、コピーして、
行の最後にカーソル合わせてエンター押して、右クリック押して、貼り付け選ぶ、
って作業してる人もいるんじゃないかなぁと思います。
(もちろんキーボードショートカットを使えば幾分はマシですが)
じゃぁこういう時Vimはどうするのかというと、
yypと入力すると同じ動作をします。
「え、キー入力したら文字入力されないの？」と思ったあなた、Vimにはキーを入力した時、
コマンド(ショートカットのようなもの)として認識されるモードと、
普通に入力されるモードが分かれているんです。
これをうまく切り替える事によってVimは作業効率の改善をすることができます。
早速コマンドの解説に入ります。
さっき説明したコマンドはyypでしたが、これはyyというコマンドとpというコマンドと分けて認識されます。
yyというコマンドは現在の行をコピーするというコマンドで、
pというコマンドはコピーしたものを貼り付けるというコマンドです。
yと言うのはヤンク、pと言うのはペーストを意味していると思えば覚えやすいですね。
ちなみに切り取りはyの代わりにcを押すことで実現できます。カットのcです。
詳しくコマンドを説明するときりがないので詳しくはドットインストールの動画や、既存のチュートリアルサイト
http://mattn.kaoriya.net/software/vim/20110902125512.htmや、
http://vim-jp.org/vimdoc-ja/usr_toc.htmlなどをみるといいと思います。
int main() {
~~~~~~
~~~~
~~
}というコードの~~で書かれた部分(複数行ありますがどこでもいいです)
を全部消すには~~の何処かにカーソルがあるときにci}と入力してみてください、
全部なくなります。
"abcde"という文字列を修正したい時はci"と打てば、
abcdeが消えて""の中にカーソルが入った状態で入力できるモードになります。
文字がいくら増えてもできます。これがVimの強みです。
いくら文字数が変わっても同じコマンドを打てば同じように操作できる、これが大きな特徴です。
慣れてきたらVimにプラグインを入れたり、vimrcやgvimrcと呼ばれるものを弄ったりしてみましょう。
Twitterが出来るようになったり、IDEみたいに文字列の補完ができたり(Sublime Textのような変数名補完も可能です)、
シンタックスハイライトやチェックが出来たりもします。
do-whileやswitch文の文法忘れちゃった...ってなっても大丈夫です。
スニペット補完というものを使えば例えばswitchと打ってあるキーを押すと、
switch (a) {
  case :
    
    break;
 default:

  break;
}
と展開されます。実際には補完が優秀なのでswiとか押すだけでも大丈夫だったりします。
コード書くの楽に思えてきたんじゃないかと思います。
